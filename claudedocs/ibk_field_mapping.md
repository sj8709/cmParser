| 행 | C0 (라벨) | C1 (값) | C2 (라벨) | C3 (값) |
|---|---|---|---|---|
| 0 | `직책1)` | **직책명** | `성명` | **이름** |
| 1 | `직위2)` | **직위** | `현 직책 부여일 \n (임원 선임일 등)` | **날짜** (예: `2025. 03. 26.`) |

**후처리** (설정 §4 표B row 4~):

- 개행 포함 원문은 그대로 보존 (`[법령] ... | [내규] ...`)

## 4. 표C (관리의무) — 1행 × 1열, 텍스트 블록

- 단일 셀 다단락 구조
- **bold 단락 = 책무명 제목** (엑셀 B열로)
- **non-bold 단락 = 세부 관리의무 항목** (엑셀 C열, `- ` 접두사 + 개행 결합)

예시 (T2):
```
[B] 내부통제등의 전반적 집행 및 운영에 대한 최종적 책임     ← 제목1
    이사회에서 수립한 내부통제등 정책에 따라, ...             ← 세부1-1
    임직원이 법령 및 내부통제기준등을 준수하기 위하여 ...     ← 세부1-2
    ...
[B] 임원 직책별로 책무를 적절하게 배분한 책무구조도의 마련 및 관리에 대한 책임   ← 제목2
    지배구조법에 따라 내부통제등 관리의무를 ...                ← 세부2-1
    ...
```

### 4.1 고유책무 vs 공통책무
- **마지막 블록**이 "소관 조직의 내부통제등 관리조치의 수립·운영 및 이행에 대한 책임" 계열 → **공통책무**
- 나머지 모두 → **고유책무**
- **예외**: 대표이사 시트는 전부 **고유책무** (공통책무 없음)

### 5.4 행 조정 규칙 (설정 §5)
- **뒤에서 앞으로** 순서: 관리의무 → 책무 → 회의체
- 데이터 > 기본행 수 → 마지막 데이터행 아래 행 삽입
- 데이터 < 기본행 수 → 초과분 삭제
- 회의체 0개 → 1행만 남기고 `N/A`
- 행 삽입 시 인접행 서식 상속 — 헤더 서식 전파 주의
- 행 삭제 후 데이터 범위 밖 잔존 셀 `.clear(contents)` (서식은 유지)

### 5.5 셀 서식
- 모든 데이터 셀에 `numberFormat='@'` (텍스트) — `-`로 시작하는 값 수식 해석 방지
- 복제 직후 `B5:D(각주행)` 범위 `.clear(contents)` 먼저 수행 (원본 잔존 데이터 제거)

## 6. 검증 규칙 (설정 §10)
1. 전 시트 셀 스캔 → `#NAME?`, `#VALUE!`, `#REF!` 0건
2. 관리의무 C열 `- ` 시작 줄 수 == DOCX 원본 non-bold 단락 수
3. 각 섹션 데이터끝행 ~ 각주행 사이에 빈 셀만 존재 (잔존 0건)
4. DOCX 표B 데이터행 수 == 엑셀 책무내용 행 수

## 7. Classifier 보강 필요 지점

현재 `classifier.py`에 없지만 IBK 파싱 필요한 라벨 정규식:
- `EXEC_INFO`: `직책1?\)?`, `직위2?\)?`, `겸직여부3?\)?`, `겸\s*직\s*사\s*항4?\)?`, `소관부서`, `주관회의체5?\)?`
- `RESP_SUMMARY`: `책무\s*개요1?\)?`, `책무\s*배분일자2?\)?`
- `RESP_CONTENT`: `책무3?\)?`, `책무\s*세부내용4?\)?`, `관련\s*법령\s*및\s*내규5?\)?`
- `COMMITTEE` (중첩): `회의체명`, `위원장/위원`, `개최주기`, `주요\s*심의·의결사항`
- `OBLIGATION`: **라벨 없음** — 표C는 `rows=1, cols=1` + bold 단락으로 식별

현 `tests/test_classifier.py` 10개 테스트는 위 라벨 변형을 이미 커버하므로 정규식만 `\s*`, `\)?` 등 약간 완화하면 됨.
